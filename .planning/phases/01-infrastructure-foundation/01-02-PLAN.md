  <name>Task 1: Create GitHub Actions CI workflow</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create `.github/workflows/ci.yml` -- a custom workflow (NOT VGV reusable workflows per CONTEXT.md research fallback -- VGV workflows pull in very_good_cli + bloc_tools unnecessarily).

Use the template from 01-RESEARCH.md as the base, with these requirements:

**Trigger:** on push to main, on pull_request to main.

**4 Jobs:**

1. **`analyze`** -- Format check + static analysis
   - Matrix: flutter-version `["3.19.0", ""]` (empty string = latest stable, per CONTEXT.md locked decision)
   - Steps: checkout@v4, flutter-action@v2 (with flutter-version from matrix, channel: stable, cache: true), flutter pub get, `dart format --set-exit-if-changed lib test`, `flutter analyze --no-fatal-infos`

2. **`test`** -- Run test suite
   - Matrix: flutter-version `["3.19.0", ""]`
   - Steps: checkout@v4, flutter-action@v2, flutter pub get, `flutter test`
   - Note: Coverage reporting is not needed yet (no real tests). Add `--coverage` flag in Phase 2 when actual tests exist.

3. **`publish-check`** -- Verify package is publishable
   - No matrix needed (latest stable only)
   - Steps: checkout@v4, flutter-action@v2 (channel: stable, cache: true), flutter pub get, `dart pub publish --dry-run`

4. **`pana`** -- Package analysis scoring
   - No matrix needed (latest stable only)
   - Steps: checkout@v4, flutter-action@v2 (channel: stable, cache: true), `dart pub global activate pana`, `dart pub global run pana --no-warning --exit-code-threshold 120`
   - Threshold 120 (not 160) because Phase 1 won't have example/ or dartdoc coverage. Full 160 is a Phase 5 gate.

**Important details:**
- Use `subosito/flutter-action@v2` (locked in CONTEXT.md)
- Use `actions/checkout@v4`
- Enable `cache: true` on flutter-action for faster CI
- For the matrix with empty string (""), subosito/flutter-action treats this as "latest stable" -- this is the documented behavior
- Do NOT use `--no-fatal-infos` on analyze in a way that hides real warnings. The flag converts info-level diagnostics from fatal to non-fatal, which is correct (VGA marks some info-level hints).

After creating the workflow, validate the YAML syntax locally:
```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))"
```
